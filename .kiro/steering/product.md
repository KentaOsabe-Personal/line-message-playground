# プロダクト概要

LINE Message Playground は、個人開発者が LINE Messaging API による配信を、実アカウントを使った小さく安全な構成から段階的に学ぶためのローカル検証環境です。

最初から本番向けの配信管理機能を網羅するのではなく、自分宛ての送信を再現可能かつ追跡可能にし、その後に必要な能力だけを追加します。

## 対象ユーザー

- LINE Messaging API の送信・認証・Webhook・運用制約を実践的に学ぶ個人開発者
- 学習用 LINE 公式アカウントを自分で管理し、まず自分自身を送信対象にできる利用者

複数事業者向けの運用、大規模キャンペーン、高度な顧客管理は初期スコープに含めません。

## 中核能力

### 開発環境の疎通確認

フロントエンド、バックエンド、データベースを一体で起動し、ブラウザからバックエンド API までの接続状態を確認します。

### 安全なテスト配信

最初は自分宛ての固定テキストによる push、または自分だけが友だちの broadcast を扱います。配信対象と内容を送信前に確認できることを重視します。

### 秘密情報の隔離

チャネルアクセストークンやチャネルシークレットはバックエンドだけが参照します。ブラウザ、React、API レスポンス、Git、ログへ露出させません。

### 冪等性と監査可能性

配信は取り消せない外部作用として扱います。一意キーによる二重送信防止と、対象・内容・日時・結果・LINE のリクエスト ID を追跡できる設計を基本とします。

### 段階的な機能拡張

一方向配信だけなら Webhook を導入しません。個別ユーザーの収集や応答が必要になった段階で、公開 HTTPS、署名検証、イベント重複排除を追加します。

## 主要ユースケース

- ローカル環境を起動し、Frontend から Backend API への接続を確認する
- Django 経由で自分の LINE アプリへテストメッセージを送り、実機表示と API 結果を確認する
- 送信前の確認と冪等化により、連打や再試行による二重配信を防ぐ
- 配信履歴と利用量を確認し、失敗や月間上限を安全に扱う
- 必要に応じて送信方式、Webhook、ユーザー ID 管理へ学習範囲を広げる

## 価値提案

- 実際の LINE Messaging API の制約を、UI とバックエンドの両方で学べる
- 最小の成功体験から始め、秘密管理、冪等性、監査、個人データ保護へ段階的に進める
- 機能数ではなく、安全に再現でき、結果を追跡できることを成功の基準にする

## プロダクト原則

- 学習に不要な LINE ユーザーデータは保存しない
- ユーザー ID を保存する場合は、目的・アクセス権・保持期間・削除方法を決める
- 対象人数、料金プラン、月間上限を送信前に確認できるようにする
- LINE プラットフォーム経由の負荷試験や、不特定多数への試験配信を行わない
- 外部 API の失敗、429、タイムアウトを正常系とは別の明示的な状態として扱う
